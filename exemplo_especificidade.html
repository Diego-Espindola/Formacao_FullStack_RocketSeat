Cheat sheet — Especificidade CSS

Guarde este arquivo para consultar rapidamente a ordem de prioridade, exemplos e as observações sobre !important e :not().

Resumo rápido (do menor para o maior)

  • Seletores universais * (mais fraco)
  • Seletores de tipo (elementos) — div, p
  • Classes, atributos e pseudo-classes — .classe, [attr], :hover
  • IDs — #meuID
  • Estilo inline — style=""
  • !important — exceção: ganha de todas as regras normais (ver observações)

Tabelinha de especificidade (formato: a,b,c,d)

SeletorExemploEspecificidade
Inlinestyle=""1,0,0,0
ID#meuID0,1,0,0
Classes / atributos / pseudo-classes.classe, [type="text"], :hover0,0,1,0
Tipo / elementos / pseudo-elementosdiv, p, ::before0,0,0,1
Universal*0,0,0,0

Regras: Some as partes para seletores combinados. Se empatar, o último declarado vence.

Observações importantes

  • !important não altera especificidade — ele simplesmente faz a declaração ganhar de outras sem !important. Se houver empate entre dois !important, aplica-se a especificidade normal e depois a ordem (último declarado vence).
  • :not() não acrescenta ao contador de pseudo-classes. Porém, o conteúdo dentro de :not(...) conta como se estivesse fora.
  • Evite depender de !important — prefira tornar o seletor mais específico ou reorganizar o CSS.

Exemplo visual

HTML: 
<div id="meuID" class="pai">
  <p class="classe">Alvo de exemplo</p>
</div>

<p class="classe importante">Exemplo com !important</p>
Como o navegador decide (ordem aplicada):
  1. Regras de tipo (p) — especificidade 0,0,0,1.
  2. Regras de classe (.classe) — especificidade 0,0,1,0.
  3. Regras por ID (#meuID p) — especificidade 0,1,0,0.
  4. Inline — style="..." — especificidade 1,0,0,0.
  5. !important vence todas as regras sem !important.
Simulação (visual):

Este parágrafo é afetado por p, .classe e #meuID.

Este outro parágrafo tem .importante { color: ... !important }

Exemplos práticos e notas finais

  • div p (dois seletores do tipo) = 0,0,0,2 (soma das partes de elemento).
  • #id .classe[type="text"]:hover = 0,1,2,0 (1 ID + 2 em classes/atributos/pseudo-classes).
  • a:link:not(.externo) — :not() não acrescenta, mas .externo dentro conta: 0,0,1,1.
  • Dica: se estiver brigando com regras herdadas, abra o DevTools, selecione o elemento e veja a coluna "Especificidade" (ou calcule os números) e a regra que venceu).
